# hiddink-harness

> **다중 AI 에이전트 서비스를 위한 범용 하네스(Universal Harness)**
>
> `hiddink-harness`는 Claude Code, agy, gpt-codex, Kimi 등 다양한 AI 개발 에이전트들이 단일 프로젝트 내에서 충돌 없이 평화롭게 작동할 수 있도록 지원하는 초경량 범용 하네스 및 오케스트레이션 엔진입니다.

---

## 🎯 핵심 기능

1. **다중 프로바이더 공존**: 각 AI 에이전트 특성에 맞춤 설계된 디렉토리(`.claude/`, `.agy/` 등) 및 전용 플러그인 사양 자동 대응.
2. **SSOT 기반 동적 배포**: 중앙 소스에서 에이전트 정의, 스킬, 행동 규칙을 단 한 번만 관리하고, 필요한 프로바이더별 포맷에 맞춰 실시간 배포.
3. **글로벌 다국어 지원**: 개발자별 선호 언어(한국어, 영어 등) 설정을 반영하여 로컬 환경에 다국어 에이전트 템플릿 렌더링.
4. **Stdio 기반 MCP 내재화**: 네트워크 포트 충돌 위험이 전혀 없도록 Stdio 기반 자식 프로세스로 격리된 MCP 서버를 구성하여 보안 및 성능 극대화.

## 🗂️ 배포 컴포넌트 현황

아래 파일 개수와 카운트는 `templates/manifest.json`과 동기화되어 관리됩니다.

### Agents (49)

각 파일은 단일 전문가 에이전트를 정의합니다. 프론트매터 설정을 기반으로 최적의 모델과 전용 도구 세트가 자동 바인딩됩니다.

### Skills (121)

재사용 가능한 개발 도구 및 리팩토링 스킬 묶음입니다. 에이전트가 필요에 따라 임포트하여 작업을 자동 수행할 수 있습니다.

### Rules (23)

에이전트가 코딩 작업 시 반드시 준수해야 하는 행동 지침 및 품질 검증 룰(MUST/SHOULD/MAY)입니다.

### Guides (57)

클라우드 네이티브 설계, 토큰 효율화, 보안 수칙 등 에이전트가 작업을 수행하면서 참고하는 표준 레퍼런스 문서입니다.

## 🚀 빠른 시작 (Quick Start)

### 1. 전역 설치
```bash
npm install -g hiddink-harness
```

### 2. 프로젝트 초기화
```bash
cd your-project
hiddink-harness init --yes
```

`hiddink-harness init`은 선택된 언어와 환경에 맞추어 `CLAUDE.md`, `.claude/`, `.agy/` 폴더 구조를 자동으로 구성 및 배포합니다.
